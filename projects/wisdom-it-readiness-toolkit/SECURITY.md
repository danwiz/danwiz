# Security Policy

## Supported status

The toolkit is currently an incubating governed working draft and is not yet a production security product.

## Reporting a vulnerability

Do not include secrets, personal data, client records, or exploit details in a public issue. Report concerns privately to danewisdom@gmail.com with the subject `Wisdom IT Readiness Toolkit Security`.

Include the affected component, version or commit, reproduction conditions, expected impact, and a safe proof of concept where appropriate.

## Security principles

- collect the minimum evidence required;
- keep assessment data local by default;
- use deterministic scoring for core results;
- separate untrusted source content from executable instructions;
- require explicit authorization for destructive, external, or public actions;
- avoid storing credentials or sensitive client evidence in the repository.
