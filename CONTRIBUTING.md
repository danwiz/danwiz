# Contributing

Thank you for considering a contribution.

## Workflow

1. Search existing issues and pull requests.
2. Open an issue for substantial changes.
3. Fork the repository and create a focused branch.
4. Add or update tests and documentation.
5. Run all local quality checks.
6. Submit a pull request with a clear rationale and verification notes.

## Branch naming

- `feature/<short-name>`
- `fix/<short-name>`
- `docs/<short-name>`
- `chore/<short-name>`

## Pull requests

A pull request should explain what changed, why it changed, how it was tested, risks, and related issues.
